- 접수 자동전송 URL 변경

  - 알림 메일 발송에 실패해도 접수 자체는 시트에 남도록 구성되어 있습니다.

  - `schedule.items`
- FAQ 수정
  - `faq`

## 수정할 때 원칙

- `신고 완료 후 기재 예정`으로 둔 `평생교육시설 신고번호`는 실제 신고 전까지 바꾸지 않습니다.
- 공식 운영 이메일이 확정되기 전에는 임의 이메일을 넣지 않습니다.
- `enroll.gasUrl`이 비어 있으면 홈페이지는 자동 접수 대신 복사/문자/이메일 수동 전달 화면을 보여줍니다.
- `enroll.turnstileRequired`를 `true`로 쓸 때는 `enroll.turnstileSiteKey`와 Apps Script의 `TURNSTILE_SECRET_KEY`를 같이 넣어야 합니다.
- 원본 데이터는 `Enrollments` 탭에 쌓이고, 실제 운영 확인은 `접수관리` 탭에서 보면 됩니다.
- `접수관리` 탭이나 `접수보안로그` 탭이 없어져도 `gas-backend.js` 기준으로 다시 만들 수 있게 정리되어 있습니다.
- 중복 접수 차단이나 보안 보강을 바꿀 때는 `ieumedu.or.kr/gas-backend.js`를 Apps Script에 다시 붙여넣고 재배포해야 실제 반영됩니다.
- 입력값은 기본적인 수식 인젝션, 과도한 링크 입력, 숨김 필드 봇 입력을 방어하도록 정리되어 있습니다.
- 브라우저 임시저장은 만료시간이 지나면 자동으로 지워지며, 중복확인에 필요한 최소 정보만 남기도록 구성되어 있습니다.
- 배포된 웹앱 차단 동작은 `npm run test:enroll:gas -- <웹앱 URL>`로 다시 확인할 수 있습니다.
- 중복 접수 차단까지 같이 확인하려면 `DUPLICATE_EMAIL`, `DUPLICATE_PHONE`, `DUPLICATE_COURSE` 환경변수를 함께 넣고 테스트합니다.
- 배열 항목을 복사해서 같은 형식으로 추가하면 됩니다.

## 보안 운영 원칙

- 접수용 Google 계정은 개인 메인 계정과 분리해서 사용합니다.
- 접수용 Google 계정에는 2단계 인증과 가능하면 패스키를 설정합니다.
- Google Sheet 공유는 링크 공개를 끄고, 편집자는 최소 인원만 남깁니다.
- Apps Script 편집 권한도 최소 인원만 유지하고, 외부 협업자 추가 시 만료 일정을 함께 관리합니다.
- Turnstile를 실제 운영에 적용할 때는 `turnstileSiteKey`와 `TURNSTILE_SECRET_KEY`를 동시에 설정하고 테스트 후 `turnstileRequired`를 `true`로 바꿉니다.
- `접수보안로그` 탭에서 차단 기록, 알림 실패, 비정상 반복 요청이 늘어나는지 주기적으로 확인합니다.
- 접수 시트나 Google 계정에 이상 로그인, 공유 변경, 비정상 앱 권한이 보이면 즉시 비밀번호/세션을 정리하고 공유 설정을 재점검합니다.
- 해킹이나 유출이 의심되면 먼저 공유 중지, Apps Script 배포 중지, 접속경로 차단, 비밀번호 변경, 로그 보존을 진행합니다.
- 실제 유출이 확인되면 개인정보보호법상 72시간 내 통지·신고 기준을 공식 안내에 따라 바로 검토해야 합니다.
